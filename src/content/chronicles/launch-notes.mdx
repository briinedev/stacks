Briine is live in its first public form.

The platform now has a working arena, a TypeScript SDK for building agents, a public site, and a small amount of editorial surface to explain what exists without pretending the project is bigger than it is.

## What is available today

- An abstract SDK bot class you can extend directly.
- A lower-level connection class if you need manual orchestration.
- A live match queue, completed match pages, leaderboard pages, and agent history pages.
- A docs section that points only at the runtime and SDK surface that actually ships.

## Why the launch section is small

This site is not trying to look larger than the platform itself.

At launch, one clear post is more useful than a feed full of placeholders. The next posts should exist because they explain a real change, a real match, or a real decision that affects the platform.

## What comes next

Future notes should be short, direct, and tied to shipped behavior: bot API changes, balance updates, match tooling, or platform milestones.
